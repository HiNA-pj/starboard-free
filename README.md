# Starboard Free

OBS Studio で使えるリアルタイム勝敗カウンターです。
操作画面で WIN / LOSE をカウントすると、OBS のブラウザソースにリアルタイムで反映されます。
一般の配信者でもダブルクリックで簡単に起動できる設計になっています。

---

## 概要

- **配信向けの勝敗カウンター**: 配信画面に重ねて表示できるシンプルなスコアボードです。
- **OBS overlay 対応**: 背景自動透過で、配信画面に違和感なくオーバーレイできます。
- **ローカルサーバーによるリアルタイム同期**: 操作画面のクリックとOBS表示画面が即座に連動します。

## はじめに必要なもの

Starboard Free を使うには **Node.js** が必要です。Node.js をインストールすると、`npm` コマンドも使えるようになります。まだ Node.js がインストールされていない場合は、[Node.js 公式サイト](https://nodejs.org/)からインストールしてください（推奨は LTS 版です）。

配布物（ZIP）には `node_modules`（依存ファイル）は含まれていません。初回起動前に、**`server` フォルダで `npm install` を実行**してください。

```bash

npm install


cd server
npm install
```

この手順は初回のみ必要です。2回目以降は `start-starboard.bat` をダブルクリックするだけで起動できます。

> 💡 Node.js がインストールされているか確認するには、ターミナルで `node -v` と入力してください。バージョン番号が表示されれば問題ありません。
>
> ℹ️ **開発者向け補足**: `dist` ビルドを作り直す場合は、Starboard Free のルートフォルダで `npm install` と `npm run build` を実行してください。通常のユーザーはこの操作は不要です。

## 各画面のURL

Starboard Free には2つの画面があります。役割の違いを理解しておきましょう。

- **操作画面**: `http://localhost:3001/`
  - 配信者がWIN / LOSEのカウント操作や設定変更を行う画面です。
  - ブラウザで開いて操作します。
- **OBS Overlay 画面**: `http://localhost:3001/overlay`
  - OBSのブラウザソースに追加する表示専用のURLです。
  - この画面自体を直接操作することはできません。操作画面での変更が自動的に反映されます。
  - OBSを起動しなくても、ブラウザで開いて表示を確認できます。OBS設定前のプレビューにご利用ください。

## OBS Studio への設定方法

1. OBS Studio を開きます。
2. ソース一覧の「＋」ボタンをクリックし、「ブラウザ」を選択します。
3. 新しいソースを作成します（名前は任意）。
4. 以下の設定を入力します。

| 項目 | 設定値 |
|---|---|
| **URL** | `http://localhost:3001/overlay` |
| **幅** | 選択した表示モードの推奨サイズ（下記参照） |
| **高さ** | 選択した表示モードの推奨サイズ（下記参照） |

> 💡 OBS の「ブラウザソース」では「ローカルファイル」ではなく **URL を指定**してください。上記の URL（`http://localhost:3001/overlay`）を入力します。

### 表示モードと推奨サイズ

お好みに応じて表示サイズ（レイアウト）を2種類から切り替えられます。
操作画面で切り替えると、OBS上の推奨サイズ表示も自動的に連動します。

- **Standard (通常表示)**:
  - 推奨サイズ: **幅 800 / 高さ 200**
  - 配信画面でしっかりと目立たせたい場合におすすめ。
- **Compact (小さめ表示)**:
  - 推奨サイズ: **幅 480 / 高さ 120**
  - 配信画面の隅に邪魔にならないように置きたい場合におすすめ。余白を抑え、可読性を重視したレイアウトです。

> 💡 **表示モード設定は維持されます**: スコアを「リセット」しても、設定した表示モード（Standard/Compact）はリセットされずそのまま維持されます。

## カラープリセット (Color Preset)

操作画面から、オーバーレイの背景・文字・枠線などの配色テーマを一発で切り替えることができます。
Free版では以下の6つの固定プリセットから選択可能です（カラーピッカーや自由編集は未実装です）。
Standard / Compact の両レイアウトで共通のプリセットが適用されます。

1. **Default**:
   - 既存の紫/ピンク系のネオンテイスト。
2. **Blue**:
   - 青・シアン系。落ち着いていて配信画面を邪魔しない爽やかな配色。
3. **Red**:
   - はっきりとした赤系。勝負感のある配色。
4. **Mono**:
   - 白・グレー系。どんな配信画面にも合わせやすいシンプルで汎用的な無彩色。
5. **Sky**:
   - 明るい水色系。爽やかで配信画面に映える配色。
6. **Pink**:
   - ピンク系。華やかで目を引く配色。

> 💡 **カラープリセットは維持されます**: スコアを「リセット」しても、設定したカラープリセットは維持されます。

## Undo（1つ戻す）

WIN / LOSE を誤って押してしまった場合、直前のカウント操作を1回だけ戻せます。

- WINを押した直後：UndoでWINが1つ戻ります
- LOSEを押した直後：UndoでLOSEが1つ戻ります
- Undoできるのは直前の1回だけです
- Reset後はUndoできません
- タイトル変更、表示モード変更、カラープリセット変更はUndo対象外です
- Reset自体をUndoする機能ではありません

## 連勝・連敗表示 (Streak)

現在の連勝または連敗の数を、オーバーレイと操作画面に小さく表示します。

### 表示される場所

- **Standard Overlay**: 勝率の右側に小さく表示されます。
- **ControlPanel**: 勝率の横に表示されます。
- **Compact Overlay**: 現在は非表示です（レイアウトを優先）。

### 表示例

- `3連勝中` — 3回連続でWINしている状態
- `2連敗中` — 2回連続でLOSEしている状態

### 仕様

- **連勝中**: 正の数値で表示（`1連勝中` / `2連勝中` / ...）
- **連敗中**: 負の数値で表示（`1連敗中` / `2連敗中` / ...）
- **なし**: `0` のときは表示されません
- **Resetで消える**: スコアをリセットするとStreakも `0` になり非表示になります
- **Undoで戻る**: 直前のWIN/LOSE操作をUndoすると、そのときのStreak状態に戻ります
- **履歴は保存しない**: 最高連勝記録や過去のStreak履歴は保存されません。今この配信の流れだけを一時的に表示します。

### 動作例

1. Reset → Streak非表示
2. WIN → `1連勝中`
3. WIN → `2連勝中`
4. LOSE → `2連敗中`（連勝が切れて連敗に）
5. Undo → `2連勝中`（LOSEを戻す）
6. Reset → Streak非表示

## 結果をコピー (Session Summary)

配信後・練習後に、現在の対戦結果を1クリックでクリップボードにコピーできます。

### 使い方

ControlPanel の Live Control セクション内にある「結果をコピー」ボタンをクリックすると、現在の対戦結果がクリップボードにコピーされます。

### コピー内容

- **Title**: 設定したタイトル（デフォルトの「勝敗カウンター」の場合は省略）
- **勝敗数**: 現在のWIN数とLOSE数
- **勝率**: 現在の勝率

### コピー文例

**Titleがある場合（例：「ランクマ」）:**
```
今日の対戦結果：ランクマ
12勝8敗 / 勝率60%
```

**Titleがデフォルトの場合:**
```
今日の対戦結果：
12勝8敗 / 勝率60%
```

### 仕様

- **0勝0敗でもコピー可能**: スコアが0の状態でもボタンは有効です
- **currentStreakは含めない**: 連勝・連敗情報はコピー文に含まれません
- **履歴保存なし**: コピーした結果は保存されません。毎回手動でコピーする必要があります
- **自動投稿なし**: SNSへの自動投稿機能はありません
- **テンプレ編集なし**: コピー文のカスタマイズはできません

## 同期と動作の仕組み

- **ローカルAPIサーバーによる同期**: 
  OBSのブラウザソースと操作画面ブラウザのlocalStorageはセキュリティの都合上共有されないため、本ツールはバックグラウンドで動作するローカルAPIサーバー (`http://localhost:3001`) を介して瞬時に状態を同期しています。
- **接続ステータスの表示**: 
  操作画面ヘッダーの右側に現在の接続状態が表示されます。
  - 🟢 **サーバー同期中** (緑色、`Wifi` アイコン): サーバーと正常に通信・同期が行われています。
  - 🟡 **ローカル動作中** (黄色、`WifiOff` アイコン): サーバー未起動時。この状態でもローカルのlocalStorageを使い、操作画面単体で勝敗カウンターとして機能します。

---

## 注意点

- **使用ポートについて**: 本ツールは `3001` のポートを使用します。すでに他のアプリでこのポートを使用している場合、ポート競合が起きる可能性があります。
- **サーバー未起動時**: サーバーを起動していない状態で `http://localhost:3001/overlay` を開いても、OBSへの同期は行われません。必ず `start-starboard.bat` などでサーバーを起動した状態にしてください。

## トラブルシューティング / Known Issues

### `npm install` が失敗する

- Node.js がインストールされているか確認してください。ターミナルで `node -v` を実行し、バージョン番号が表示されなければ、[Node.js 公式サイト](https://nodejs.org/)からインストールしてください。
- ネットワーク環境によっては、`npm install` が依存パッケージをダウンロードできない場合があります。その場合は時間を置くか、別のネットワークをお試しください。

### 黒い画面（ターミナル）を閉じてしまった

`start-starboard.bat` を再度ダブルクリックしてください。黒い画面を閉じるとサーバーが停止し、操作画面とOBS Overlayの同期が止まります。使用中は必ず起動したままにしてください。

### サーバー未接続表示（🟡 ローカル動作中）が出ている

- `start-starboard.bat` が起動しているか確認してください。
- `server` フォルダで `npm install` が済んでいるか確認してください。
- 改善しない場合は、`start-starboard.bat` を一度閉じてから再度起動し直してください。

### ポートが競合している

本ツールは `3001` のポートを使用します。他のアプリがこのポートを使っている場合は、そのアプリを終了してから再度お試しください。

### OBSのブラウザソースに正しく表示されない

- ソースの種類は「ブラウザ」を選択し、**「ローカルファイル」ではなくURLを指定**してください。
- URLは `http://localhost:3001/overlay` を設定してください。
- APIサーバーが起動していないとOBS側に操作画面の変更が反映されません。`start-starboard.bat` が起動していることを確認してください。

### 企業PCや制限環境について

社内ネットワークやセキュリティ制限の厳しい環境では、localhostへの通信がブロックされ正常に動作しない場合があります。個人の配信環境でのご利用を推奨します。
