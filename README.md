# Starboard Free

OBS Studio で使えるリアルタイム勝敗カウンターです。
操作画面で WIN / LOSE をカウントすると、OBS のブラウザソースにリアルタイムで反映されます。
一般の配信者でもダブルクリックで簡単に起動できる設計になっています。

---

## 概要

- **配信向けの勝敗カウンター**: 配信画面に重ねて表示できるシンプルなスコアボードです。
- **OBS overlay 対応**: 背景自動透過で、配信画面に違和感なくオーバーレイできます。
- **ローカルサーバーによるリアルタイム同期**: 操作画面のクリックとOBS表示画面が即座に連動します。

---

## 起動方法

### Windowsの場合 (推奨)

1. GitHubなどからZIPでダウンロードした場合は、**必ずZIPファイルを解凍**してからご利用ください。ZIPを開いただけの状態では正しく起動できません。
2. 解凍したフォルダ内の **`start-starboard.bat`** をダブルクリックするだけで、APIサーバーとViteが同時に起動します。

> 💡 **初回起動時のみ**: 起動する前に一度だけ、プロジェクトのフォルダでターミナルを開き、`npm install` を実行してください。`server` フォルダでも同様に `npm install` が必要です。

> ⚠️ **起動中は黒いターミナル画面を閉じないでください**: `start-starboard.bat` を実行すると黒いターミナル画面が開きます。これはStarboardを動かすための画面です。使用中は閉じずにそのままにしておいてください。閉じるとControlPanelやOBS Overlayの同期が停止します。最小化しておいても問題ありません。

### コマンドで手動起動する場合
PowerShellやターミナルを開き、以下のコマンドを実行します。
```bash
npm run dev:all
```

## 各画面のURL

Starboard Free には2つの画面があります。役割の違いを理解しておきましょう。

- **操作画面**: `http://localhost:5173/`
  - 配信者がWIN / LOSEのカウント操作や設定変更を行う画面です。
  - ブラウザで開いて操作します。
- **OBS Overlay 画面**: `http://localhost:5173/overlay`
  - OBSのブラウザソースに追加する表示専用のURLです。
  - この画面自体を直接操作することはできません。操作画面での変更が自動的に反映されます。
  - OBSを起動しなくても、ブラウザで開いて表示を確認できます。OBS設定前のプレビューにご利用ください。

## OBS Studio への設定方法

1. OBS Studio を開きます。
2. ソース一覧の「＋」ボタンをクリックし、「ブラウザ」を選択します。
3. 新しいソースを作成します（名前は任意）。
4. 以下の設定を入力します。

| 項目 | 設定値 |
|---|---|
| **URL** | `http://localhost:5173/overlay` |
| **幅** | 選択した表示モードの推奨サイズ（下記参照） |
| **高さ** | 選択した表示モードの推奨サイズ（下記参照） |

### 表示モードと推奨サイズ

お好みに応じて表示サイズ（レイアウト）を2種類から切り替えられます。
操作画面で切り替えると、OBS上の推奨サイズ表示も自動的に連動します。

- **Standard (通常表示)**:
  - 推奨サイズ: **幅 800 / 高さ 200**
  - 配信画面でしっかりと目立たせたい場合におすすめ。
- **Compact (小さめ表示)**:
  - 推奨サイズ: **幅 480 / 高さ 120**
  - 配信画面の隅に邪魔にならないように置きたい場合におすすめ。余白を抑え、可読性を重視したレイアウトです。

> 💡 **表示モード設定は維持されます**: スコアを「リセット」しても、設定した表示モード（Standard/Compact）はリセットされずそのまま維持されます。

## カラープリセット (Color Preset)

操作画面から、オーバーレイの背景・文字・枠線などの配色テーマを一発で切り替えることができます。
Free版では以下の6つの固定プリセットから選択可能です（カラーピッカーや自由編集は未実装です）。
Standard / Compact の両レイアウトで共通のプリセットが適用されます。

1. **Default**:
   - 既存の紫/ピンク系のネオンテイスト。
2. **Blue**:
   - 青・シアン系。落ち着いていて配信画面を邪魔しない爽やかな配色。
3. **Red**:
   - はっきりとした赤系。勝負感のある配色。
4. **Mono**:
   - 白・グレー系。どんな配信画面にも合わせやすいシンプルで汎用的な無彩色。
5. **Sky**:
   - 明るい水色系。爽やかで配信画面に映える配色。
6. **Pink**:
   - ピンク系。華やかで目を引く配色。

> 💡 **カラープリセットは維持されます**: スコアを「リセット」しても、設定したカラープリセットは維持されます。

## Undo（1つ戻す）

WIN / LOSE を誤って押してしまった場合、直前のカウント操作を1回だけ戻せます。

- WINを押した直後：UndoでWINが1つ戻ります
- LOSEを押した直後：UndoでLOSEが1つ戻ります
- Undoできるのは直前の1回だけです
- Reset後はUndoできません
- タイトル変更、表示モード変更、カラープリセット変更はUndo対象外です
- Reset自体をUndoする機能ではありません

## 同期と動作の仕組み

- **ローカルAPIサーバーによる同期**: 
  OBSのブラウザソースと操作画面ブラウザのlocalStorageはセキュリティの都合上共有されないため、本ツールはバックグラウンドで動作するローカルAPIサーバー (`http://localhost:3001`) を介して瞬時に状態を同期しています。
- **接続ステータスの表示**: 
  操作画面ヘッダーの右側に現在の接続状態が表示されます。
  - 🟢 **サーバー同期中** (緑色、`Wifi` アイコン): サーバーと正常に通信・同期が行われています。
  - 🟡 **ローカル動作中** (黄色、`WifiOff` アイコン): サーバー未起動時。この状態でもローカルのlocalStorageを使い、操作画面単体で勝敗カウンターとして機能します。

---

## 注意点

- **使用ポートについて**: 本ツールは `5173` (フロント) および `3001` (APIサーバー) のポートを使用します。すでに他のアプリでこれらのポートを使用している場合、ポート競合が起きる可能性があります。APIサーバー（`localhost:3001`）は通常ユーザーが直接開く必要はありません。
- **サーバー未起動時**: サーバーを起動していない状態で `http://localhost:5173/overlay` を開いても、OBSへの同期は行われません。必ず `start-starboard.bat` などでサーバーを起動した状態にしてください。
